À propos du projet

Présentation

Application web en Go pour explorer les Pokémon via l'API PokeBuild.

Fonctionnalités : recherche, filtres, pagination, favoris.

FAQ - Gestion de projet

Comment avez-vous décomposé le projet ?

Phases : Choix de l'API → Structure du projet → Fonctionnalités de base → FT1 à FT4 → Tests et documentation.

Comment avez-vous organisé le travail ?

Projet individuel. Todo list, priorités sur les fonctionnalités obligatoires, développement itératif.

Comment avez-vous géré votre temps ?

Priorités : structure de base, puis FT1-FT4, puis pages requises, puis gestion erreurs, enfin documentation.

Quelle stratégie pour vous documenter ?

Documentation PokeBuild API, documentation Go (net/http, html/template), exemples en ligne.

Informations techniques

Technologies : Go (librairies standard), HTML, CSS

API : PokeBuild API (https://pokebuildapi.fr/api/v1)

Endpoints :

  • GET /api/v1/pokemon - Liste complète
  • GET /api/v1/pokemon/{id} - Détails
  • GET /api/v1/pokemon/type/{type} - Par type
